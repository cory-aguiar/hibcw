Supabase Database Reference
Admin & Superadmin Only
Schema v2 · PostgreSQL via Supabase · All tables use UUID primary keys
1
Table Overview
incidents
All reported incidents — type, location, time, description, suspect & vehicle details, map coordinates.
Public: SELECT (active), INSERT · Admin: UPDATE, DELETE
incident_media
Photos and videos attached to incidents. Linked via incident_id . Files stored in Supabase Storage.
Public: SELECT, INSERT · Admin: DELETE
incident_notes
Admin/Captain internal notes attached to specific incidents. Not visible to the public.
Auth only: SELECT, INSERT, DELETE
business_registry
Business membership applications and approved members. Tracks status, approval, and packet delivery.
Public: INSERT · Auth: SELECT, UPDATE, DELETE
member_applications
Legacy member application table. New registrations use business_registry .
Public: INSERT · Auth: SELECT, UPDATE, DELETE
member_emails
Approved member email list used to verify identity on incident submissions. Unique constraint on email.
Public: SELECT · Auth: INSERT, DELETE
district_alerts
District-level alerts sent by Captains. Displayed as banners on the public map. Supports media attachments.
Auth only: SELECT, INSERT, UPDATE, DELETE
admin_roles
Maps authenticated Supabase users to roles: superadmin , admin , captain . Links to auth.users .
Auth only: SELECT, INSERT, UPDATE, DELETE
Storage: incident-media
Public Supabase Storage bucket. Files stored at path incidents/<id>/<filename> .
Public: SELECT, INSERT · Auth: DELETE
incidents primary table
Column Type Notes
id uuid PK · auto-generated
created_at timestamptz Submission timestamp, defaults to now()
type text Incident category — required
type_color text red · gold · teal — controls map pin color
occurred_at timestamptz When incident happened — required
address text Street address or intersection — required
description text Reporter's narrative — required
reporter_name text? Optional — many reports are anonymous
police_report text? HPD report number if filed
suspect_height … suspect_other text? 7 optional suspect description fields
vehicle_make … vehicle_notes text? 6 optional vehicle description fields
lat / lng float8? Map pin coordinates — may be null
status text active (default) · archived
admin_roles authentication
Column Type Notes
id uuid PK
user_id uuid FK → auth.users(id)
role text superadmin · admin · captain
email / full_name text? Display info — denormalized from auth
district text? Captain's district ID (e.g. south-hilo )
last_sign_in_at timestamptz? Updated on each login
district_alerts captain alerts
Column Type Notes
district text District ID (e.g. north-hilo )
message text Alert body text — required
alert_type text emergency · bolo · amber · incident · pattern · scam · trespass · welfare · allclear · info — default: incident
media_url / media_type text? Optional attached image or video
is_edited bool True if alert was updated after posting
sent_by_email / name text? Captain who sent the alert
archived_at timestamptz? Set when archived — null means publicly visible. Archived alerts are hidden from the public feed but retained permanently as a record
archived_by text? Email of the admin or captain who archived the alert — audit trail for proof of concept reporting
business_registry membership
Column Type Notes
business_number text? Assigned on approval (e.g. D3-042 )
status text pending · approved · rejected · archived
district text? District ID auto-detected from address
captain_name text? Assigned at approval
submitted_at timestamptz? Application received date
approved_at timestamptz? Approval date — null until approved
packet_sent_at timestamptz? Welcome packet generation date

Row Level Security (RLS) Summary
Public (unauthenticated) access
SELECT
Active incidents, incident media, member_emails (for verification)
INSERT
New incidents, incident media, new business applications
BLOCKED
All other operations on all other tables
Authenticated (admin / captain) access
FULL
incidents (update, delete, view archived), incident_notes, admin_roles, district_alerts
FULL
business_registry, member_applications, member_emails
DELETE
incident_media, storage objects in incident-media bucket
