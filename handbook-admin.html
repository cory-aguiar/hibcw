1. Understanding Admin Roles

HIBCW uses three distinct roles in the admin panel. Understanding the difference is essential for managing the platform effectively.

FeatureAdminSuperadmin
View all incidents✅ All districts✅ All districts
Archive / restore incidents✅ All districts✅ All districts
Delete incidents permanently✅ Yes✅ Yes
Bulk archive / delete✅ Yes✅ Yes
Add / remove verified members✅ Yes✅ Yes
Edit member profiles✅ Yes✅ Yes
Import member CSV✅ Yes✅ Yes
Approve / reject applications✅ All districts✅ All districts
Delete applications✅ Yes✅ Yes
Assign district captains✅ Yes✅ Yes
Upload captain photos✅ Any district✅ Any district
Send district alerts✅ Any district✅ Any district
Delete district alerts✅ Yes✅ Yes
Export CSV (incidents/members)✅ Yes✅ Yes
Invite new admins / captains❌ No✅ Yes
Remove admin accounts❌ No✅ Yes
Edit admin roles & districts❌ No✅ Yes
Access Manage Admins screen❌ No✅ Yes
💡 Key Distinction

Admins and Superadmins have near-identical access to day-to-day operations. The only actions exclusive to Superadmins are those that affect admin accounts themselves — inviting new users, removing accounts, and changing roles. This ensures the admin roster can only be managed by the most trusted users.

2. Signing In

Admins access the panel at your deployment URL with "/admin.html" appended (e.g., hibcw.vercel.app/admin.html). A mobile-optimized version is also available at "/admin-mobile.html".

  • Enter your email address and password and click "Sign In →"
  • If you've forgotten your password, click "Forgot password?" to receive a reset link
  • On mobile devices, visiting admin.html will prompt you to switch to the mobile version automatically
🔑 Account Setup

Admin accounts are created by a Superadmin via the Manage Admins screen. You cannot self-register. If you need access, contact your HIBCW Superadmin and provide your email address and the role you require.

3. The Admin Interface

The desktop admin panel uses a fixed left sidebar for navigation and a main content area on the right. The mobile version uses a bottom tab bar with six tabs. Both surfaces connect to the same live Supabase database.

3.1 Desktop Navigation (admin.html)
  • 🚨 Active Incidents — currently reported, active incidents
  • 📦 Archived Incidents — resolved or removed from the public map
  • 📋 All Incidents — combined view with status column
  • 📈 Statistics — incident counts, type breakdown, district breakdown
  • 🗺 Map View — interactive leaflet map of all incidents
  • 📍 Districts — captain cards, directory, alert tools
  • 👥 Members — verified member management (admin-only tab)
  • 📋 Applications — membership application review and approval
  • 🔐 Manage Admins — invite/remove admin accounts (superadmin-only tab)
3.2 Mobile Navigation (admin-mobile.html)
  • 🚨 Incidents — searchable incident list with tap-to-detail
  • 📊 Stats — stat tiles and type/district breakdown bars
  • 👥 Members — member cards (admin-only tab)
  • 📋 Apply — application review with pending badge count
  • 📍 Districts — captain overview, directory, and alert tools
  • ☰ More — account info, CSV exports, theme toggle, sign out
3.3 Real-Time Updates

Both panels subscribe to Supabase real-time channels. New incidents and district alerts appear instantly without requiring a page refresh. A live indicator dot in the header turns teal when connected and gold when reconnecting.

4. Incident Management
4.1 Viewing & Searching Incidents

The Active, Archived, and All Incidents views each show a sortable table of incidents. Use the search bar to search across type, address, description, and reporter name.

  • Active — incidents currently visible on the public map
  • Archived — incidents removed from public view but retained in the database
  • All — combined view with a Status column; useful for cross-status searches
4.2 Filtering Incidents

Each incident view has an advanced filter panel accessible via the "⚙ Filters" button. Filters are additive — multiple filters narrow results simultaneously.

  • Type — filter by incident category (Robbery, Burglary, Assault, etc.)
  • Date Range — filter by custom start and end dates
  • District — filter to a specific HIBCW district
  • Radius Search — enter an address and radius in miles to find nearby incidents

Active filters appear as dismissible chips below the toolbar. Click the ✕ on any chip to remove that filter, or use "Clear All" to reset.

4.3 Archiving, Restoring & Deleting
  • 📦 Archive — removes the incident from the public map without deleting it. Use this for resolved incidents, confirmed duplicates, or reports under review.
  • ↩ Restore — returns an archived incident to active status and the public map.
  • 🗑 Delete — permanently removes the incident and all associated media files from the database and storage. This cannot be undone.
⚠️ Delete is Permanent

Deleting an incident removes it from the database forever, including all uploaded photos and videos. Always archive first unless you are certain the report is invalid. When in doubt, archive — not delete.

4.4 Bulk Actions

On the Active incidents view, you can select multiple incidents using the checkboxes on the left of each row.

1
Select Incidents
  • Check the box next to each incident you want to act on
  • Or use the header checkbox to select all visible rows
2
Choose Action
  • The bulk action bar appears showing the count of selected items
  • Click "📦 Archive Selected" or "🗑 Delete Selected"
3
Confirm
  • Confirm when prompted — the action applies to all selected incidents
💡 Tip

Use the type and date filters to narrow down a set of incidents before using bulk select. For example, filter to a specific type from a specific date range, then bulk archive all results at once.

4.5 Incident Detail & Notes

Click "View" on any incident row (or tap any card on mobile) to open the full detail panel. This shows:

  • Date, time, and age tag
  • District and assigned captain contact card
  • Address with Google Maps Street View link (if GPS coordinates exist)
  • Full description, reporter name, police report number
  • Suspect description (height, build, age, gender, race, clothing, other)
  • Vehicle description (make, model, color, year, plate, notes)
  • Attached media (photos and videos, click to open lightbox)
  • Follow-up notes log — add new notes or delete existing ones

Notes are visible to all admin and captain users and are never shown on the public site.

4.6 Exporting Incidents to CSV

Each incident view has an "⬇ Export CSV" button in the toolbar. The export includes all visible rows (after any active filters) with full incident, suspect, vehicle, and GPS fields. Files are named automatically, e.g., HIBCW_active_2026-03-14.csv.

5. Member Management

The Members screen manages the list of verified HIBCW members. Verified members receive a "✅ Verified Member" badge on any incidents they report through the public site.

5.1 Viewing & Searching
  • Search by name, email, phone, business name, city, state, or zip code
  • Filter by district using the district dropdown
  • Each row shows: name, email, phone, business, address, district, captain, date added, and days active
5.2 Adding Members

You can add a verified member directly by email without requiring them to submit an application.

1
Enter Email
  • Type the member's email in the "Enter email address to add…" field
2
Add to List
  • Press Enter or click "＋ Add Member"
  • The email is added to the verified member list immediately
  • Their reports will show the verified badge from this point forward
💡 Note

Adding a member by email creates a basic verified record. To attach a full profile (name, business, address, district), either ask the member to submit an application and approve it, or use the Edit function to fill in the details manually.

5.3 Importing Members via CSV
  1. Click "⬆ Import CSV" in the Members toolbar
  2. Select a CSV file where each row contains an email address in the first column
  3. The system adds all emails not already in the list — duplicates are silently skipped
5.4 Editing Member Profiles

Click "✏️ Edit" on any member row to open the full edit form. You can update: full name, email, phone, business name, address, city, state, zip, district assignment, captain name, and profile photo (up to 4 MB).

5.5 Removing Members

Click "Remove" on any member row to remove them from the verified members list. This removes their verified status — future reports from that email will no longer show the member badge. Their past reports are not affected.

5.6 Exporting Members to CSV

Click "⬇ Export CSV" in the Members toolbar. The export includes: number, full name, email, phone, business name, address, city, state, zip, district, captain, date added, and days active.

6. Application Management

The Applications screen shows all membership applications submitted through the public "Join HIBCW" form. Applications can be filtered by status: Pending, Approved, or Rejected.

6.1 Reviewing an Application
  • Each card shows: applicant name, business name, address, email, phone, district, notes, and submission date
  • Pending applications have a gold left border; approved are teal; rejected are red
6.2 Approving an Application
  1. Find the pending application
  2. Click "✅ Approve" and confirm
  3. The applicant is added to the verified members list
  4. If the district has a captain assigned, their captain's name is automatically populated on the new member record
6.3 Other Actions
  • Reject — marks the application as rejected; the applicant is not added to the member list
  • Revoke — removes an approved member's verified status and removes them from the member email list
  • Re-approve — re-approves a previously rejected application
  • Delete — permanently removes the application record from the database
💡 District Captains Can Also Act

District Captains can approve, reject, revoke, and re-approve applications within their own district. Only admins can delete application records permanently.

7. Districts & Captains

The Districts screen is the hub for all nine HIBCW districts. It has two tabs — Overview and Directory — and includes tools to manage captains, upload photos, and print rosters.

7.1 District Overview & Stats
  • Captain photo, name, business, phone, and email
  • Four stat tiles: member count, incidents this month, this week, and today
  • Last reported date and top-4 active incident type breakdown bars
  • "👥 Members" button — opens the full member roster for that district
  • "👮 Set Captain" button — opens the captain assignment modal (admin-only)
7.2 Assigning District Captains
1
Open Set Captain
  • Click the "👮 Set Captain" button on a district card in the Overview tab
2
Select the Captain
  • A modal opens showing all verified members in that district
  • Search by name, email, or business using the search bar
  • Click the row of the member you want to assign as captain
3
Save
  • Click "Save Captain" to confirm
  • The captain's name is written to all member records in that district
  • The district card updates immediately with the new captain's details
💡 Tip

The captain must be a verified member of that district to appear in the selection list. If the intended captain is not yet a member, add them first, set their district, then assign them as captain.

7.3 Captain Photo Upload
  1. In the district Overview tab, hover over the captain's photo or initials avatar
  2. A "📷 Change" or "📷 Upload" overlay appears — click it
  3. Select a JPG, PNG, or WebP image up to 4 MB
  4. The photo uploads to storage and the district card updates immediately
7.4 Captain Directory & Print

The Directory tab shows a searchable list view of all 9 districts. Click the "🖨 Print PDF" button to generate a print-optimized landscape PDF directory of all captains.

7.5 District Member Roster

From any district card, click "👥 View Members" to open a modal showing all verified members in that district. A "🖨 Print Roster" button generates a formatted PDF roster for that district.

8. District Alerts

District alerts allow admins and captains to send messages directly to the community. Alerts appear on the public-facing website and mobile app within seconds of being sent.

8.1 Sending an Alert
  1. Click the "📢 Send District Alert" button in the Districts toolbar
  2. Select the target district from the dropdown
  3. Type your message
  4. Optionally attach a photo or video using the 📎 Attach File button
  5. Click "📤 Send Alert"
8.2 Editing an Alert

In the alert log, click the "✏️ Edit" button on any alert to load its content back into the composer. Update the message or media and click "💾 Save Changes". Edited alerts show an "edited" tag publicly.

8.3 Deleting an Alert

Click the "✕" button on any alert in the log to permanently remove it. If the alert had media attached, the media file is also deleted from storage.

8.4 Where Alerts Appear Publicly
  • A slide-up banner on the map page (desktop and mobile) for alerts from the last 48 hours
  • The "📢 Alerts" tab in the side panel (desktop) or the More screen (mobile)
  • Alerts are real-time — visitors see new alerts without refreshing
9. Map View

The Map View screen shows all active incidents plotted on an interactive Leaflet map. It is available to all admin roles and provides a geographic perspective on community activity.

9.1 Map Controls
  • STREET / SATELLITE — toggle between street and satellite base maps
  • CLUSTER — toggle marker clustering (groups nearby incidents into numbered circles)
  • HEATMAP — toggle a heat layer showing incident density across the island
  • DISTRICTS — toggle district boundary overlays with color-coded polygons
9.2 Map Sidebar & Popups

The right sidebar lists all currently mapped incidents. Click any card to pan the map and highlight its marker. Click any map marker to open a popup — click "View Details" to open the full incident detail modal without leaving the map.

10. Statistics

The Statistics screen gives a high-level overview of incident data, filterable by district.

10.1 Stat Tiles
  • Active, Archived, Total, Last 30 Days, With GPS Pin, With Media
10.2 Type & District Breakdown

A bar chart showing active incidents by type, sorted by frequency. A second chart shows incidents by district — clicking a district bar applies that district as a filter across the stats view.

11. Manage Admins Superadmin Only

The Manage Admins screen is only accessible to Superadmins. It allows you to view all current admin and captain accounts, invite new users, and modify or remove existing accounts.

11.2 Inviting a New Admin or Captain
1
Fill in the Invite Form
  • Enter the new user's email address
  • Select a role: Admin, Captain, or Superadmin
  • If inviting a Captain, select their assigned district from the dropdown
2
Send the Invite
  • Click "✉️ Send Invite"
  • The system creates the account and sends a "Set your password" email
  • The invite link is valid for a limited time — prompt new users to act promptly
3
Confirm
  • The new account appears in the admin table
  • The user can now sign in at admin.html with the password they set
11.3 Editing & Removing Accounts

Click "✏️ Edit" on any account row to update the full name, role, and district assignment. Click "Remove" to revoke admin access — note that you cannot remove your own account.

⚠️ You Cannot Remove Your Own Account

The Remove button is hidden on the row that matches your own email address. To remove your own access, have another Superadmin do it, or contact your HIBCW technical administrator.

13. Tips & Best Practices
Incident Hygiene
  • Review the Active incidents queue daily during busy periods — stale reports degrade the public map's usefulness
  • Archive resolved incidents promptly and add a note explaining the resolution before archiving
  • Use the radius filter to check for clustering in a neighborhood or business district
  • Reserve permanent deletion for clearly invalid or test submissions only
Member Management
  • Approve applications from the Applications screen rather than manually adding emails — it captures the full business profile
  • After approving an application, check that the district and captain name fields populated correctly on the member record
  • Run an occasional export and review for outdated or duplicate entries
District Alerts
  • Only send alerts for actionable, district-relevant information — over-alerting reduces community engagement
  • Include media (photo or video) where available to increase credibility and urgency
  • Cancel outdated alerts so the public feed stays current
Account Security (Superadmins)
  • Limit Superadmin accounts to the smallest number of people who genuinely need account management access
  • Review the admin accounts table periodically and remove inactive accounts
  • Captain accounts should be scoped to a single district — always verify the district assignment when inviting a new captain
Light & Dark Mode
  • Both admin panels support light and dark mode via the ☀️/🌙 toggle in the header (desktop) or the More tab (mobile)
  • The preference is saved per browser and persists across sessions
